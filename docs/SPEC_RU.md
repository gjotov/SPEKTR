# Техническая спецификация криптосистемы «СПЕКТР-26»

## 1. Общее описание
**СПЕКТР-26** — это модульная симметричная криптосистема повышенной защищенности, реализованная на языке Rust. Система специализируется на **стеганографической инкапсуляции данных** и обеспечении **правдоподобного отрицания**. Спроектирована для скрытого хранения информации, невидимого для форензик-анализа и устойчивого к физическому принуждению.

## 2. Иерархия ключей
### 2.1 Аппаратная привязка (Silicon DNA)
Система привязывает процесс шифрования к конкретному оборудованию хост-машины.
- **Источник:** UUID машины (серийные номера материнской платы/процессора).
- **Преобразование:** `Hardware_DNA = SHA-256(Machine_UID)`.
Это предотвращает несанкционированную дешифровку на любом устройстве, кроме оригинального.

### 2.2 Функция деривации ключа (KDF)
Для защиты от перебора и словарных атак используется **Argon2id**.
- **Вход:** Пароль пользователя + Hardware DNA.
- **Выход:** 512-битный поток данных, разделенный на:
  - `Cipher_Key` (256 бит)
  - `MAC_Key` (256 бит)
- **Параметры:** $m=64MB, t=3, p=1$.

## 3. Криптографическое ядро
### 3.1 Движок детерминированного хаоса
Внутренняя энтропия генерируется с помощью **Логистического отображения** в арифметике с фиксированной запятой (Q32):
$$x_{n+1} = 4.0 \cdot x_n \cdot (1 - x_n)$$
Это гарантирует идентичность хаотической траектории на разных архитектурах CPU без дрейфа вычислений с плавающей запятой.

### 3.2 Сеть замен-перестановок (SPN)
- **Динамический S-Box:** Генерируется через перемешивание Фишера-Йетса на основе хаоса. Статические таблицы отсутствуют.
- **Константно-временная замена:** Поиск в таблице реализован через битовые маски для предотвращения атак по побочным каналам (Cache-timing).
- **Многофазная диффузия (P-Box):** 128-битный слой перестановок с использованием ARX-операций (Add-Rotate-Xor) для соблюдения **строгого лавинного критерия (SAC)**.

## 4. Инкапсуляция и стеганография
### 4.1 WAV-контейнер
Система скрывает шифротекст внутри валидного аудиофайла формата **8-bit Mono LPCM WAV**.
- **Скрытие:** Зашифрованная полезная нагрузка заменяет аудиоданные, выглядя как низкоуровневые радиопомехи или белый шум.
- **Zero Metadata:** Внутри зашифрованного потока отсутствуют заголовки, магические числа или маркеры длины.

### 4.2 AEAD (Аутентифицированное шифрование)
«СПЕКТР-26» следует парадигме **Encrypt-then-MAC**:
1. **Шифрование:** Потоковый режим Chunked CTR (счетчик) с блоками по 128 бит.
2. **Аутентификация:** Тег HMAC-SHA256, вычисляемый от Nonce и шифротекста.

## 5. Протоколы защиты
### 5.1 Правдоподобное отрицание
Контейнер поддерживает архитектуру **«Двойного дна»**:
- **Decoy Slot (Приманка):** Открывается decoy-паролем, содержит ложные данные.
- **Real Slot (Скрытый):** Скрыт внутри высокоэнтропийного шума. Его наличие невозможно доказать без мастер-ключа.

### 5.2 Протокол «Выжженная земля» (Panic Protocol)
При активации **Ключа Паники** система:
1. Возвращает пользователю ложные данные.
2. Запускает фоновый **многопоточный процесс затирания (shredding)**.
3. Перезаписывает контейнер случайным шумом и удаляет файл из файловой системы.
